# Architecture

## 전체 파이프라인

```
┌─────────────────────────────────────────────────────────────────────────┐
│  dataflow-saas - B2B 데이터 분석 플랫폼                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  [데이터 소스]                                                           │
│  GitHub, Jira, CSV, API, ...                                           │
│      │                                                                  │
│      ▼                                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  1. ingestion/ - 수집                                           │    │
│  │     collector/: 다양한 소스에서 데이터 수집                       │    │
│  │     crawler/: 스케줄 기반 크롤링                                 │    │
│  └───────────────────────────────┬─────────────────────────────────┘    │
│                                  ▼                                      │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  2. transform/ - 변환                                           │    │
│  │     converter/: 파일 포맷 변환 (Excel, JSON → Parquet)           │    │
│  │     tagger/: 메타데이터 태깅                                     │    │
│  │     llm-transform/: AI 기반 변환                                │    │
│  └───────────────────────────────┬─────────────────────────────────┘    │
│                                  ▼                                      │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  3. storage/ - 저장                                             │    │
│  │     warehouse/: 데이터 웨어하우스 (DuckDB/BigQuery)               │    │
│  │     sync/: DB 간 동기화                                         │    │
│  │     catalog/: 메타데이터 카탈로그                                │    │
│  └───────────────────────────────┬─────────────────────────────────┘    │
│                                  ▼                                      │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  4. analytics/ - 가공/모델링                                     │    │
│  │     models/: 데이터 모델 (SRCH → S → H → RPT)                    │    │
│  │     dataform/: SQL 템플릿 (dbt 스타일)                           │    │
│  │     incremental/: 증분 처리 로직                                 │    │
│  └───────────────────────────────┬─────────────────────────────────┘    │
│                                  ▼                                      │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  5. visualization/ - 시각화                                      │    │
│  │     chart-api/: 차트 데이터 API                                  │    │
│  │     dashboard/: 대시보드 UI (선택)                               │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 데이터 모델 흐름 (analytics/)

```
원본 데이터 (TAG_PREP)
    │
    ▼
SRCH_* ─────────────────── 조회: 필요한 컬럼만 가져옴
    │
    ▼
S_* ────────────────────── 가공: 필터, 집계, 축 변환
    │
    ▼
H_* ────────────────────── 계산: 테이블 간 연산 (달성률 = 실적/계획)
    │
    ▼
RPT_* ──────────────────── 시각화: 차트용 최종 형식
    │
    ▼
차트 (API 응답)
```

## 기술 스택 (예정)

| 레이어 | 기술 |
|--------|------|
| 언어 | Python 3.11+ |
| API | FastAPI |
| 데이터 처리 | pandas, polars |
| 파일 포맷 | Parquet (pyarrow) |
| 데이터베이스 | DuckDB (로컬), BigQuery (클라우드) |
| 워크플로우 | Airflow (또는 간단한 스케줄러) |
| 컨테이너 | Docker |
| IaC | Terraform |

## 구현 우선순위

### Tier 1 (핵심)
1. `transform/converter` - 파일 변환 (V2 설계 기반)
2. `ingestion/collector` - 데이터 수집 (GitHub API)
3. `storage/warehouse` - 데이터 저장 (DuckDB)

### Tier 2 (확장)
4. `analytics/models` - 데이터 모델링
5. `visualization/chart-api` - 차트 API
6. `orchestration/scheduler` - 스케줄링

### Tier 3 (운영)
7. `observability/monitoring` - 모니터링
8. `auth/api-gateway` - 인증

### Tier 4 (고급)
9. `agents/ai-agent` - AI 에이전트
10. `agents/instance-manager` - 인스턴스 관리
